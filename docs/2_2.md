# 발표 내용

## 테스트와 테스트 주도 개발

* 테스트란 무엇일까?

테스트란 소프트웨어가 요구사항에 따라 정확하고 안정적으로 작동하는 지 검증하는 과정을 말한다. 테스트는 확인하지 못한 오류를 검출해내고 소프트웨어의 성능을 평가하는 등 제품의 품질을 높이는 데 기여한다.
직접 코드를 실행해보는 것 부터 작성된 시나리오에 따라 자동으로 진행하는 등 다양한 방식이 존재한다.

1. 단위 테스트와 통합 테스트

테스트를 하는 규모에 따라 단위 테스트와 통합 테스트로 나뉜다.
* 단위 테스트: Unit(단위) 별로 진행하는 테스트로 클래스나 메소드 등 작은 규모로 진행한다. 클래스나 메소드 마다 분리되어 진행되기에 각각이 빠르고 독립적으로 결과를 얻을 수 있다.
* 통합 테스트: 소프트웨어의 통합적인 부분을 확인하기 위한 테스트로 전체적인 흐름과 연결을 확인하는 것이 주요 목적이다. 많은 구성 요소가 한번에 작동하기에 느리게 진행되지만 실제 서비스를 이용하는 입장에서 확인할 수 있다는 장점이 존재한다.

이렇듯 단위 테스트와 통합 테스트는 사용 목적과 규모가 다르기 때문에 모두 사용되는 것이 일반적이다.

2. 테스트 주도 개발이란?

테스트 주도 개발, 일명 TDD는 (Test Driven Development) 테스트를 기반에 두고 개발하는 방식을 말한다. 
일반적인 개발은 '코드 개발 > 테스트' 순서로 이루어지지만 TDD는 요구사항에 맞는 테스트 코드를 먼저 개발한 이후 이에 맞추어 실제 코드를 개발한다.
또한 테스트에 맞추기 위한 개발과 개발된 코드의 리펙토링이 반복해서 진행되면서 보다 깨끗한 코드와 빠른 테스트 케이스 제작이 가능하다. 

## 자바에서의 테스트 코드 작성

* 자바에서 테스트를 위해 사용되는 메소드

자바에서 테스트를 위해선 메소드를 제작하게 되는데 이때 JUnit5를 이용할 경우 다음과 같은 어노테이션을 사용한다.
1. @Test: 테스트 메소드임을 표시
2. @BeforeEach: 테스트 실행 전, 예를 들어 현재 클래스의 @Test가 포함된 각 메소드 전에 실행
3. @AfterEach: @BeforeEach와 반대로 현재 클래스의 각 테스트 메서드 실행 후 실행됨
4. @BeforeAll: 모든 Test 전에 딱 한번 실행
5. @AfterAll: 모든 Test가 다 끝나고 마지막 한번 실행
이 외에도 테스트 메소드를 위한 다양한 어노테이션이 존재한다.

테스트 메소드 내부에서 검증을 위해 사용되는 메소드도 다양하게 존재한다. 
isTrue(), isEqualTo() 실행 결과값을 비교하기 위한 메소드가 종종 사용되며, 예외 처리를 위한 assertThrows(), assertThat() 등의 메소드가 사용된다.

테스트에 사용될 가짜 객체 생성을 위한 Mockito 프레임워크도 존재한다. 스피링에서 JUnit과 결합해 사용할 수 있다.

    * Service Layer의 테스트
    * Controller Layer의 테스트
