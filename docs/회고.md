# 셈웨어 인턴쉽 멘토링 프로그램 - To Do List 만들기
작성자: `nicola`
## 1. 개요
11월 21일부터 약 4주간의 시간 동안 셈웨어 학생 인턴을 대상으로 멘토링 프로그램이 진행되었습니다.
멘티는 스프링 부트를 바탕으로 주차별 과제를 수행하며 간단한 JPA 기반의 To Do List 서버를 개발했습니다.
멘티의 다양한 학습 수준을 고려하여 서버-클라이언트 구조부터 스프링 시큐리티까지의 주제를 학습 하였습니다.
스프링 멘토링 프로그램을 통해서 백엔드의 여러 기본 개념과 구현 능력을 가질 수 있었습니다.

## 2. 과제와 구현
### 2.1 1주차 1차시: spring framework
본격적인 백엔드 개발 진행에 앞서서 서버-클라이언트 구조와 spring 등장 배경을 통해 spring의 전체적인 특징에 대해 학습였습니다..
spring framework의 등장은 개발자들이 기본 Java 문법으로 쉽게 서버 프로그램을 개발할 수 있는 환경을 마련 했습니다.
spring boot는 이러한 spring에 필요한 설정을 준비해주어 현재까지 많은 서버 개발이 spring으로 진행 됩니다.

해당 과제에서 가장 중요했던 부분은 Servlet이었던 것 같습니다. DI, IOC, AOP 등의 개념도 물론 중요하지만, 이후 통신을 이해하기 위해선 Servlet이란 개념의 이해가 필요했기 때문입니다.
### 2.2 1주차 2차시: JPA와 Database
서버를 구성하기 이전 준비해야하는 것 중 하나는 데이터 구조를 구성하는 것입니다. 
이를 위해서 DB, 그중에서도 MySQL에 대해 배우고 Spring Data JPA의 개념과 사용법에 대해 학습하였습니다.
작성한 ERD를 바탕으로 Entity class를 구성하기도 했습니다.

직접 ERD을 그리면서 DB의 구성에 대해 많은 고민을 가지게 되었었습니다. 
여러 요소를 고려해 ERD를 작성했지만 결국 개발이 진행되면서 Entity의 요소를 여러번 수정할 수 밖께 없었습니다.

### 2.3 2주차 1차시: API와 Controller&Service
웹서비스에서 서버와 클라이언트의 통신은 주로 http rest api에 의해 이루어집니다. 
따라서 To Do List에 필요한 기능을 생각하고 API를 구현하였습니다.

이 부분은 모든 주제 중 가장 쉬웠던 부분이었습니다. 팀프로젝트에서 http api 를 다룰 기회가 종종 있었기 때문이었기 때문입니다.
### 2.4 2주차 2차시: 테스트 코드와 TDD
대량의 작업을 수행하는 서버 서비스에서는 오류를 찾기 위한 테스트가 매우 중요합니다. 
TDD는 이러한 테스트를 먼저 만들고 충족시키는 방식인데 이미 개발이 진행되었기 때문에 테스트 코드만 작성 하였습니다.

TDD로 개발하지 않더라고 테스트 코드는 항상 중요합니다. 
사람이 일일이 다 테스트할 수도 없거니와 필요할 때마다 자동으로 테스트가 가능하기 때문입니다.
### 2.3 3주차: Spring Security
Spring Security는 상대적으로 긴 분량을 가졌기 때문에 예외로 1주간의 시간이 주어졌습니다.
멘티는 JWT의 생성 과정을 공부하고 직접 spring 프로젝트에 Security를 적용하였습니다.

긴 분량만큼 가장 어려웠던 부분이기도 합니다. 
실제 구현에서도 filter chain에 대한 이해가 꼭 필요했으며 버전과 문법에서 문제가 발생하기도 하였습니다.
하지만 그 만큼 인증이라는 필수적인 부분을 다루고 있어서 필요했고 다양한 보안 영역에 신경을 기울일 필요가 느껴지기도 했습니다.

## 3. 종합적으로
이번 멘토링을 통해 실제로 작동 가능한 To Do List 서버를 구현할 수 있었습니다.
특히 Jwt의 원리를 배우고 직접 구현까지 할 수 있어 어렵지만 도움이 되는 경험이었습니다.
다른 백엔드 개발을 하게 되면 좋은 코드까진 아니더라도 구현 자체에는 자신감이 생긴 것 같습니다.

스터디에서 좋았던 것은 역시 멘토의 존재였습니다. 
물론 gpt와 개발 블로그 덕분에 여러 정보를 모을 수 있었지만, 이 것들이 궁금한 점을 명쾌하게 해결해주진 않았습니다.
사실은 쉬운 개념에도 막힐 때가 있는데 이 부분을 뚫는 데 멘토님이 많은 도움이 되었습니다.

또 한가지 기억에 남는 것은 gitignore와 같은 세팅의 중요성이었습니다.
spring boot와 jpa, intellij 덕분에 매우 편하게 개발하였지만 세팅을 스스로 깔끔하게 하지 못한 것 같습니다.
yml 파일을 git에 그대로 올리거나 폴더 구조가 명확하지 않은 등 여러 아쉬움이 남았습니다.
그래도 이번 스터디에서 가지고 가는 아쉬움을 추후 할 지도 모르는 프로젝트에 반영할 수 있으리라 기대하고 있습니다.
